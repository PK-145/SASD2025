## a) Utility Class
**คลาสตัวอย่าง:** `System.Web.HttpUtility`
*(จาก System.Web Namespace)*

### การวิเคราะห์
* **ลักษณะการออกแบบ:** คลาสนี้ประกอบไปด้วย Method ที่เป็น `static` ทั้งหมด เช่น `HtmlEncode`, `UrlDecode` และ `ParseQueryString`
* **เหตุผลที่จัดอยู่ในกลุ่มนี้:** * **Helper Functions:** มันถูกสร้างมาเพื่อเป็น "เครื่องมือ" ในการจัดการกับข้อความบนเว็บ (Web strings) โดยที่เราไม่ต้องสร้าง Object (Instance) ของมันขึ้นมาใช้งาน
    * **Stateless:** คลาสนี้ไม่มีการเก็บข้อมูลส่วนตัว (State) ไว้ภายในตัวมันเอง รับค่า Input เข้าไป ประมวลผล และคืนค่า Output ออกมาทันที
    * **Global Access:** ออกแบบมาเพื่อให้เรียกใช้ได้จากทุกที่ใน Application อย่างสะดวก

#### Ref: https://learn.microsoft.com/en-us/dotnet/api/system.web.httputility?view=net-10.0

## b) Mutable Class
**คลาสตัวอย่าง:** `System.Text.Json.Nodes.JsonObject`
*(จาก System.Text.Json.Nodes Namespace)*

### การวิเคราะห์
* **ลักษณะการออกแบบ:** เป็นคลาสที่ใช้แทนโครงสร้าง JSON object ที่สามารถ "เขียน" หรือ "แก้ไข" ได้ (Writeable DOM)
* **เหตุผลที่จัดอยู่ในกลุ่มนี้:** * **State Modification:** เราสามารถเพิ่ม Property (`.Add`), ลบ (`.Remove`) หรือแก้ไขค่าใน Key ต่างๆ ได้โดยตรงหลังจากสร้าง Object ขึ้นมาแล้ว
    * **In-place Update:** เมื่อมีการแก้ไข ข้อมูลจะถูกเปลี่ยนภายใน Instance เดิมในหน่วยความจำ ไม่ได้สร้าง Object ใหม่ทุกครั้งที่มีการเปลี่ยนค่า ซึ่งเหมาะสำหรับงานที่ต้องสร้างหรือแก้ไขโครงสร้าง JSON แบบไดนามิก

#### Ref: https://learn.microsoft.com/en-us/dotnet/api/system.text.json.nodes.jsonobject?view=net-10.0

## c) Immutable Class
**คลาสตัวอย่าง:** `System.Collections.Immutable.ImmutableList<T>`
*(จาก System.Collections.Immutable Namespace)*

### การวิเคราะห์
* **ลักษณะการออกแบบ:** เป็น Collection ที่ถูกออกแบบมาให้ "ไม่สามารถเปลี่ยนแปลงได้" หลังจากถูกสร้าง
* **เหตุผลที่จัดอยู่ในกลุ่มนี้:** * **Read-Only Nature:** เมื่อเราสร้าง `ImmutableList` ขึ้นมาแล้ว เราไม่สามารถเข้าไปเปลี่ยนค่าใน Index ใดๆ หรือเพิ่ม/ลดสมาชิกใน Instance เดิมได้
    * **Return New Instance:** หากเราเรียก Method อย่าง `.Add()` หรือ `.Remove()` คลาสจะไม่ไปแก้ข้อมูลใน List เดิม แต่จะทำการสร้าง **Instance ใหม่** ที่มีข้อมูลตามที่แก้ไขแล้วคืนกลับมาให้แทน
    * **Predictability:** ช่วยลดข้อผิดพลาดในโปรแกรมที่ทำงานแบบขนาน (Concurrency) เพราะมั่นใจได้ว่าข้อมูลใน Object นี้จะไม่ถูกแอบเปลี่ยนโดยส่วนงานอื่น

#### Ref: https://learn.microsoft.com/en-us/dotnet/api/system.collections.immutable.immutablelist-1?view=net-10.0

## สรุปเปรียบเทียบ

| ประเภทคลาส | ตัวอย่างคลาส | การใช้งานหลัก | การจัดการหน่วยความจำ |
| :--- | :--- | :--- | :--- |
| **Utility** | `HttpUtility` | เรียกใช้ฟังก์ชันช่วยเหลือ | ไม่ต้องจองพื้นที่ Heap (เรียกผ่าน Static) |
| **Mutable** | `JsonObject` | แก้ไขข้อมูลที่มีโครงสร้างซับซ้อน | แก้ไขข้อมูลในตำแหน่งเดิม (In-place) |
| **Immutable** | `ImmutableList<T>` | เก็บข้อมูลที่ต้องการความปลอดภัย | สร้าง Object ใหม่ทุกครั้งที่มีการเปลี่ยนแปลง |
